### 2\. Deployment Log

**Method:** Docker Compose (Manual)  
**Working Directory:** `/opt/homepage`

#### A. Environment Prep

```bash
# System Update
apt update && apt upgrade -y
apt install curl sudo -y

# Docker Installation
curl -fsSL [https://get.docker.com](https://get.docker.com) | sh
```

#### B. Directory Structure

Iâ€™m creating a specific config directory to persist layout and service definitions.

```bash
mkdir -p /opt/homepage/config
mkdir -p /opt/homepage/images
cd /opt/homepage
```

#### C. Compose Configuration (`/opt/homepage/docker-compose.yml`)

!!! note
    Port mapped to 3005 as per network plan.

```yaml
services:
  homepage:
    image: ghcr.io/gethomepage/homepage:latest
    container_name: homepage
    restart: always
    ports:
      - "3005:3000"  # External:Internal
    volumes:
      - ./config:/app/config # Configs (services.yaml, etc)
      - ./images:/app/public/images # Custom icons
      - /var/run/docker.sock:/var/run/docker.sock:ro # (Optional) For Docker stats
    environment:
      - PUID=1000
      - PGID=1000
```

#### D. Service Start

```bash
docker compose up -d
```

### 3\. Configuration Management

**Location:** `/opt/homepage/config/` _These YAML files control the dashboard layout._

|     |     |
| --- | --- |
| File | Purpose |
| `services.yaml` | Defines the clickable apps (Jellyfin, Sonarr, etc.) and their API widgets. |
| `widgets.yaml` | Top-level information (Weather, Date, CPU load). |
| `bookmarks.yaml` | Static links (External websites, Documentation). |
| `settings.yaml` | Global themes, colors, and layout providers. |

!!! warning
    When adding integrations (e.g., Sonarr API Key), ensure they are pasted correctly in `services.yaml` to enable live stats.
